Questão 5 (Mecanismos de Conexão Simultânea - MPM): Como o Apache2 gerencia requisições concorrentes de múltiplos clientes na rede?

Resposta Técnica: O Apache2 utiliza Módulos Multi-Processamento (MPMs) para ditar sua arquitetura de concorrência no Linux:

  • Prefork: Cria um processo filho exclusivo no sistema operacional para cada conexão HTTP recebida. Não utiliza threads. É extremamente estável e seguro para isolar módulos não-thread-safe antigos (como versões legadas do PHP), mas consome muita memória RAM sob alta carga de usuários concorrentes.
  • Worker: Utiliza uma arquitetura híbrida de múltiplos processos e múltiplas threads por processo. Cada conexão cliente ganha uma thread dedicada. Isso reduz drasticamente o consumo de memória em comparação ao Prefork, permitindo escalar o volume de acessos.
  • Event: É a evolução do Worker e o padrão atual. Ele foi desenhado para lidar de forma assíncrona com conexões em estado Keep-Alive (clientes conectados que não estão solicitando dados no exato momento). Ele separa uma thread dedicada exclusivamente para gerenciar a ociosidade dessas conexões, liberando os workers principais apenas para processar requisições ativas. É o mais performático para ambientes modernos.

Questão 6 (Servidores Proxy no HTTP): Qual a utilidade e o funcionamento de um Proxy Server no ecossistema da Web?

Resposta Técnica: Um servidor Proxy atua como um intermediário na camada de aplicação (Camada 7 do modelo OSI) entre o navegador do cliente e o servidor web de destino:

  • Caching (Desempenho): Armazena localmente cópias de recursos estáticos frequentemente solicitados (imagens, folhas de estilo, arquivos HTML). Requisições subsequentes idênticas são respondidas diretamente pelo proxy, economizando largura de banda da intranet e acelerando o tempo de resposta.
  • Segurança e Filtro de Borda: O proxy esconde a topologia e os IPs privados da rede interna dos computadores da bancada. Ele inspeciona os cabeçalhos HTTP de acesso e pode atuar como um firewall de conteúdo, bloqueando o tráfego para URLs não autorizadas de acordo com as diretivas corporativas.
  • Proxy Reverso: Posicionado à frente dos servidores internos para atuar como balanceador de carga (Load Balancer), distribuindo as requisições recebidas entre múltiplas máquinas de serviço e centralizando chaves de criptografia TLS/SSL.

Questão 7 (Protocolo MIME e Serviço WEB): Explique qual a correlação direta entre as extensões MIME e o funcionamento do cabeçalho HTTP Content-Type.

Resposta Técnica: O protocolo MIME (Multipurpose Internet Mail Extensions), originalmente criado para permitir anexos multimídia em e-mails, foi incorporado ao HTTP para viabilizar a transferência de arquivos ricos na Web. Quando um cliente solicita um arquivo, o Apache2 analisa a extensão (ex: .html, .png, .shtml) e consulta sua tabela local de mapeamentos. O servidor então injeta o tipo MIME correspondente na linha do cabeçalho de resposta HTTP chamado Content-Type (ex: text/html ou image/png). O navegador do cliente lê essa linha e sabe exatamente como processar e renderizar o arquivo nativamente na tela, em vez de tentar adivinhar a codificação baseando-se apenas na extensão do arquivo.
